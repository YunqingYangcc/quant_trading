# Factor Research — 因子研究行业规范

> **核心原则：先有假说，再找数据。先验逻辑，后验 IC。**
>
> WorldQuant 招聘第一题：「请从经济学原理出发，解释你为什么认为这个因子应该有效。」答不出 → 淘汰。

---

## When to Use

以下任一场景发生时，必须先过此协议：

- 计划接入新数据源（如分析师预期、龙虎榜、产业链数据）
- 计划新增因子/特征（无论通用还是赛道专属）
- 模型长期无效（Acc ~50%），怀疑缺少有效特征维度
- 想「试试看加个 xxx 数据会不会好」

**反模式**：「我觉得这个数据可能有信号，先加进去跑跑看」— 这是数据挖掘，做多少次都停在 50%。

---

## 一、因子发现 6 问门禁

每提出一个新因子/数据源，必须书面回答以下 6 问。**答不全 → 不准开工。**

```
1. 经济逻辑：这个因子为什么应该预测未来收益？
   - 必须引用金融/经济学原理或市场微观结构理论
   - 禁止：「因为别人都用」「因为 IC 高」

2. 信息层级：这个因子属于哪一层？
   - L1 量价结构 | L2 基本面 | L3 预期/情绪 | L4 另类数据

3. 与现有因子的关系：是否已知的因子变体？和现有白名单因子的预期相关性？
   - 如果是已有因子的线性组合 → 大概率不提供增量信息

4. 覆盖度：覆盖多少只股票？数据从哪年开始？
   - A 股 5000+ 只，赛道池 ~70 只，覆盖 < 30 只 → 大概率噪声

5. 更新频率：日/周/月/季？是否存在 look-ahead bias？
   - 财报数据必须对齐公告日，不能用报告期日期

6. 失效条件：什么情况下这个因子会失效？
   - 如果答不出（「永远有效」），说明你不够理解它
```

**铁律：通过 6 问后才进入数据接入和因子构建。不准跳过。**

## 二、数据源分级体系

### 2.1 信息层级（L1-L4）

```
L4 另类数据（卫星/舆情/供应链）
  ↑ 边际收益递减，噪声指数增长
L3 预期/资金流（分析师预期/北向/龙虎榜）
  ↑ 当前最大增量空间
L2 基本面（PE/PB/ROE/营收增速）
  ↑ 已部分接入（16 个 fund_* 特征）
L1 量价结构（OHLCV 衍生指标）
  ↑ 已充分挖掘（75 个技术特征, IC≈0.01-0.05）
```

**当前瓶颈**：L1→L2 已做，L3 完全空白。L3 是最大可改进维度。

### 2.2 A 股数据源接入优先级矩阵

按 **预测力 × 低相关性 × 可接入性** 三维排序：

| 优先级 | 数据源 | 经济学逻辑 | 与现有相关性 | akshare 接口 | 接入成本 |
|:------:|:-------|:----------|:-----------:|:------------|:-------:|
| **P0** | **分析师一致预期** | 市场定价反映预期，预期修正驱动价格 | 低（全新维度） | `stock_analyst_detail_em` | 低 |
| **P0** | **北向资金流向** | 聪明钱具有信息优势 | 低（全新维度） | `stock_hsgt_*` | 低 |
| P1 | 龙虎榜 | 游资/机构异动信号 | 中（与量相关） | `stock_lhb_*` | 低 |
| P1 | 融资融券余额 | 杠杆资金情绪 | 中（与量相关） | `stock_margin_*` | 低 |
| P1 | 行业景气度指标 | 赛道 β 驱动个股 α | 中 | 需自行构建 | 中 |
| P2 | 股东户数变化 | 筹码集中度 | 低 | `stock_hold_*` | 低 |
| P2 | 限售解禁 | 供给冲击 | 低 | `stock_restricted_*` | 低 |
| P2 | 机构调研 | 信息获取行为 | 低 | `stock_jgdy_*` | 低 |
| P3 | 供应链/产业链 | 上下游传导 | 低（全新维度） | 需构建图 | 高 |
| P3 | 新闻舆情 NLP | 情绪定价 | 高（噪声大） | 需 NLP pipeline | 高 |

### 2.3 不做的事

| 数据源 | 不做的理由 |
|:-------|:----------|
| 龙虎榜明细（个股逐笔） | 仅覆盖极少数股票，赛道池覆盖 < 10% |
| 分钟级/Tick 级数据 | 周/月频调仓策略不需要，引入噪声 |
| 付费数据 API | 个人散户预算约束，先用免费源验证方向 |
| 社交媒体舆情 | NLP 成本高，A 股噪声极大（水军） |

## 三、因子构建与验证协议

### 3.1 构建规范

```
新数据源接入 → 原始数据落库 → 对齐交易日 → 处理缺失值 → 因子计算 → shift(1) → 入库
```

**对齐规则（财报类数据）**：
- 财报数据用 `公告日` 对齐，不能用 `报告期日期`
- 季报数据前向填充到日频（季度内不变）
- 公告日之前的数据 = NaN（不能提前知道）

**防 look-ahead bias 检查清单**：
- [ ] 是否用了未来日期对齐？（用公告日对齐报告期数据）
- [ ] 是否 shift(1) 了？
- [ ] 停牌日是否标记为 NaN？
- [ ] 涨跌停日是否标记不可交易？

### 3.2 单因子检验标准（三门槛）

| 检验 | 标准 | 不通过 |
|:-----|:-----|:------|
| \|Pooled Rank IC\| | ≥ 0.015 | → 淘汰 |
| 10 层分组单调性 | 首尾方向一致 | → 淘汰 |
| | 与现有所有因子相关性 | < 0.7 | → 若 > 0.7，保留 IC 更高的那个 |

**注意：小股票池（6-17 只/日）不使用日截面 IR 作为判据。IR 仅作辅助参考。**

### 3.3 增量价值检验（防冗余）

新因子必须在**控制了所有现有白名单因子后**仍有显著增量 IC：

```
新因子增量 IC = partial_corr(新因子, fwd_return | 现有白名单因子)
```

增量 IC < 0.005 → 不提供增量信息，不接入。

### 3.4 多假设检验修正

每次批量筛选 N 个新因子时，使用 **Benjamini-Hochberg 修正**：

```
1. 对所有因子的 IC p-value 排序
2. 第 k 个因子的阈值 = k/N × 0.1（FDR=10%）
3. 找到最大的 k 满足 p(k) ≤ k/N × 0.1
4. 前 k 个因子通过，其余淘汰
```

**不做的**：不筛选、不修正，一股脑全丢给模型 → 这就是现在 163→138 特征噪声淹没问题。

## 四、模型训练约束

### 4.1 固定原则

| 规则 | 理由 |
|:-----|:-----|
| 固定超参数，不调参刷指标 | 调参 = 过拟合回测期 |
| 训练集 ≤ 2022，验证集 = 2023，测试集 ≥ 2024 | 防时序泄露 |
| TimeSeriesSplit，禁止 shuffle | 防未来信息泄露 |
| 每个赛道独立模型 | 赛道间因子结构不同 |
| 只使用白名单特征 | 黑名单 = 已证伪 |

### 4.2 不稳定的处理

当前模型跨 run 结果波动大（夏普 0.5~1.2）。行业标准做法：

1. **多次训练取平均**（3-5 次独立训练，取平均预测）
2. **特征数量控制**：特征数 > 100 → 用 LightGBM 自带 feature_fraction=0.6 增加随机性
3. **Bagging 平均**：训练 5 个模型，预测时 vote/avg

### 4.3 验收标准

| 指标 | 目标 | 当前 |
|:-----|:----:|:----:|
| 模型 Acc（赛道最少） | ≥ 50% | 50-53% |
| 过拟合 gap | < 0.10 | 部分赛道 0.16 |
| 月频夏普 | ≥ 1.0 | 0.5-1.2 不稳定 |
| 分层单调性 | 高分池 > 低分池 | 待验证 |

## 五、开发流程嵌入

### 5.1 与现有 Skill 的关系

```
新因子/数据源想法
    │
    ▼
factor-research（本协议）── 6 问门禁 → 通过？
    │                                    │ 不通过 → 放弃
    ▼
check-data ── 数据质量验收
    │
    ▼
add-feature ── 6 步验证流程（计算→入库→Alphalens→白/黑名单）
    │
    ▼
screen-factors ── 多假设检验修正 → 白名单更新
    │
    ▼
preprocess ── 标准化 + 去共线
    │
    ▼
train-model ── 固定参数训练 + 过拟合检测
    │
    ▼
run-backtest ── 锁定参数回测
    │
    ▼
review-phase ── 阶段验收
```

### 5.2 本协议优先级

**factor-research > add-feature > train-model**

- 未经 6 问门禁的特征 → add-feature 拒收
- 未经多假设检验修正的批量因子 → screen-factors 拒收
- 白名单外的特征 → train-model 拒收

## 六、当前状态与下一优先级

### 6.1 已完成

- L1 量价特征：75 个技术指标（ta 库）
- L2 基本面特征：16 个（akshare：PE/PB/ROE 等）
- 赛道专属特征：108 个（拥挤度、资金流、相对强度）

### 6.2 当前瓶颈

特征维度仅覆盖 L1+L2 部分。最大增量空间在 **L3 预期/资金流层**。

### 6.3 推荐接入顺序（P0 → P1）

```
第 1 步：分析师一致预期（P0）
  - 数据：EPS 预测值、预测修正方向、预测分歧度
  - 逻辑：分析师修正方向是最强的 short-term alpha 信号之一
  - 接口：akshare stock_analyst_detail_em
  - 预计新增因子：10-15 个
  - 预期效果：模型 Acc 53% → 56%+，夏普稳定 > 0.8

第 2 步：北向资金流向（P0）
  - 数据：沪股通/深股通 每日净买入
  - 逻辑：外资具有信息优势，持续流入预示正面预期
  - 接口：akshare stock_hsgt_*
  - 预计新增因子：5-8 个
  - 预期效果：叠加后 Acc 56% → 58%+

第 3 步：融资融券余额（P1）
  - 数据：融资余额变化
  - 逻辑：杠杆资金情绪指标
  - 接口：akshare stock_margin_*

第 4 步（可选）：行业景气度（P1）
  - 需要自行构建：用赛道内个股营收增速中位数 = 赛道景气代理变量
  - 当前已有 track_sentiment，可扩展
```

### 6.4 不做的事

- ❌ 同时接入 3 个以上新数据源（必须逐个验证增量价值）
- ❌ 在模型 Acc < 55% 时继续调参
- ❌ 在未通过多假设检验时加特征到白名单
- ❌ 为了刷夏普修改回测参数
