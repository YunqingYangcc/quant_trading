## 1. 系统是什么

**TrackQuant** 是一款面向个人散户/交易员的**赛道型量化辅助决策系统**。不是全市场量化平台，不做高频交易。

### 核心理念

```
你（交易员）                 系统
┌──────────────┐          ┌──────────────────────┐
│ 主观判断赛道  │          │ 数据清洗 → 特征计算   │
│ 方向（如AI）  │          │ Alphalens 因子筛选    │
│              │  协作     │ LightGBM AI 打分     │
│ 做买卖决策    │ ──────→  │ 回测验证 → 可视化    │
└──────────────┘          └──────────────────────┘
```

- **你**：决定看好哪个赛道（半导体/AI/机器人/存储/材料/电力AI）
- **系统**：告诉你赛道内哪只票最强、什么时候该买/卖、风险多大

### 当前覆盖赛道

| 赛道 | 股票数 | 说明 |
|:-----|:------:|:-----|
| 半导体 | 17 | 设备/材料/设计/制造 |
| AI | 13 | 算力/光模块/服务器 |
| 机器人 | 12 | 减速器/伺服/控制器 |
| 存储 | 11 | NAND/NOR/接口 |
| 上游材料 | 8 | 硅片/光刻胶/靶材 |
| 电力AI | 8 | 电网智能化/AI电源 |

## 2. 环境准备与启动

### 2.1 首次安装

```bash
# 1. 克隆项目
git clone git@github.com:YunqingYangcc/quant_trading.git
cd quant_trading

# 2. 后端安装
cd backend
pip install -e .
cd ..

# 3. 前端安装
cd frontend
npm install
cd ..
```

### 2.2 启动系统（两条命令，两个终端）

**终端 1 — 启动后端（API 服务）**

```bash
cd backend
uvicorn app.main:app --reload --port 8000
```

启动成功标志：
- 终端显示 `Application ready`
- 浏览器打开 http://localhost:8000/docs 看到 Swagger API 文档
- 浏览器打开 http://localhost:8000/health 返回 `{"status": "healthy"}`

**终端 2 — 启动前端（用户界面）**

```bash
cd frontend
npm run dev
```

启动成功标志：
- 终端显示 `Local: http://localhost:5173/`
- 浏览器打开 http://localhost:5173 看到系统界面

### 2.3 端口说明

| 服务 | 地址 | 备注 |
|:-----|:-----|:-----|
| 前端界面 | http://localhost:5173 | 主要使用 |
| 后端 API | http://localhost:8000 | 前端自动连接，一般不需要直接访问 |
| API 文档 | http://localhost:8000/docs | 开发调试用 |

## 7. Alpha Research 因子研究

路径：`/alpha`

### 功能

对因子进行深度分析，验证因子是否可靠：

```
赛道选择 ┌──────────┐  因子选择 ┌──────────┐
         │ 半导体   │          │ rsi_6   │
         └──────────┘          └──────────┘

📊 因子IC时序        → 该因子过去一段时间的预测能力走势
🔥 因子相关性热力图   → Top20 因子之间的相关性（过高的要警惕）
📦 分位数组合收益    → 按因子值分 5 组，看高低组收益差距
📋 因子IC统计表      → Mean IC / IR / IC>0% 比率
```

### 交易员关注点

- **IC（Information Coefficient）**：因子与未来收益的相关性，>0 表示正向预测
- **IR（Information Ratio）**：IC 的稳定性，越大越可靠
- **分位数收益**：Q5(高分) 持续跑赢 Q1(低分) → 因子有效
- 选择某个因子后，可以看到该因子历史 IC 走势

## 8. Model Factory 模型工厂

路径：`/model-factory`

### 功能

查看和管理 AI 模型：

```
模型卡片网格（4 列）
┌──────────┬──────────┬──────────┬──────────┐
│ SEMICON. │   AI     │  ROBOT   │ STORAGE  │
│ ✅ Ready │ ⚠️ Overfit│ ✅ Ready │ ✅ Ready │
│ Train R² │ Train R² │ Train R²│ Train R²│
│ 0.0512   │ 0.0891   │ 0.0766   │ 0.0623  │
│ [Retrain]│ [Retrain]│ [Retrain]│[Retrain]│
├──────────┼──────────┼──────────┼──────────┤
│ MATERIAL │ AI-POWER │          │          │
│ ✅ Ready │ ✅ Ready │          │          │
│ [Retrain]│ [Retrain]│          │          │
└──────────┴──────────┴──────────┴──────────┘

📊 特征重要性 Top10（可切换赛道查看）
📋 模型训练结果比对表（所有模型横向对比）
```

### 操作

1. **查看模型状态**：卡片显示 Train/Val/Test R²、CV Mean R²
2. **一键重训**：点击 `🔄 Retrain` 按钮，系统重新训练该赛道模型
   - 重训过程中按钮显示 "Training..."
   - 完成后自动刷新数据
3. **特征重要性**：选择某个赛道，查看 Top10 特征的贡献度排序

### 交易员关注点

- **R²（决定系数）**：模型解释了多少收益变化
  - R² > 0：模型学到了信号（越大越好）
  - R² ≈ 0：模型基本没学到信号
  - R² < 0：模型比猜还差
- **过拟合标记**：如果出现 ⚠️ Overfit，说明模型记住了历史但可能无法泛化
- **特征重要性**：看看是哪些因子在驱动打分

## 11. 交易员日常工作流

### 每日开盘前（15 分钟）

```
Step 1 → Dashboard 首页
  看 4/6 赛道景气度 → 哪个赛道景气最高（绿色）？
  看 Quick Stats → 数据是否正常？

Step 2 → Alpha Workstation
  切换到景气最高的赛道
  看左侧 AI 排名 → Top 3 是哪几只？
  点击 Top 1 → 看 K 线形态和位置
  看右侧因子面板 → 哪些因子在驱动打分？

Step 3 → Alpha Research（每周 1-2 次）
  看因子 IC 时序 → 主要因子最近 IC 是否衰减？
  看分位数收益 → 高低分组差距是否还在？

Step 4 → Model Factory（每月 1 次）
  检查各赛道 R² → 是否有模型退化？
  需要时点击 Retrain

Step 5 → Backtest Lab（调参后或每周）
  选均衡预设 → 跑回测 → 看夏普/回撤
  如有改进 → 记录参数
```

### 模型重训触发时机

- 每月固定重训一次
- 新增了数据后（拉取了新的日线数据）
- 发现模型打分明显偏离常识时

## 12. 关键指标解读

### 因子指标

| 指标 | 读法 | 理想值 |
|:-----|:-----|:-------|
| **IC** | 因子与未来收益的相关系数。正数=正向因子 | \|IC\|≥0.02 |
| **IR** | IC 均值/标准差，衡量稳定性 | ≥ 0.5 |
| **分组收益** | 按因子值分 10 组，看高低组差距 | Q1 < Q2 < ... < Q10 |

### 模型指标

| 指标 | 读法 | 理想值 |
|:-----|:-----|:-------|
| **R²** | 模型解释了多少收益变化 | > 0（最好是 > 0.1） |
| **Train R²** | 训练集上的表现 | 比 Test 高但差距 < 0.15 |
| **Test R²** | 测试集（未见过数据）上的表现 | 接近 Train R² |
| **过拟合** | Train 远好于 Test | 差距 < 0.15 ✅ |

### 回测指标

| 指标 | 读法 | 验收底线 |
|:-----|:-----|:---------|
| **夏普比率** | 风险调整后收益 | ≥ 1.2 |
| **最大回撤** | 历史最大亏损 | < 25% |
| **年化收益** | 折合成年化的收益率 | > 10% |
| **胜率** | 盈利交易占比 | > 50% |
| **分层收益** | 高分跑赢低分 | 单调递增 |

## 13. 常见问题

### Q: 页面打开是空白的？
A: 确认后端和前端都启动了（终端 1 和终端 2 都在运行）。F5 刷新。

### Q: 数据显示为 0？
A: 检查后端终端是否有报错。可能是数据库需要重建或后端服务未启动。

### Q: Quick Stats 全 0？
A: 后端没连上或者数据库为空。检查 `http://localhost:8000/health` 返回是否正常。

### Q: 点击重训没反应？
A: 重训需要 10-30 秒，期间按钮会显示 "Training..."。如果长时间没返回，检查后端终端是否有错误日志。

### Q: 跑回测报错？
A: 需要模型已训练。先去 Model Factory 页面确认模型状态，如果显示 "模型未训练"，点击 Retrain。

### Q: 模型 R² 很低？
A: 这是当前系统的已知瓶颈——量价特征的信息量有限。预期的 Test R² 在 0~0.05 之间，不意味着系统完全无效，只是信号较弱。

### Q: 如何添加新股票？
A: 联系开发人员处理。需要更新 universe.csv、拉取数据、重算特征、重筛因子、重训模型——整套流程。

### Q: K 线图上有些线看不清？
A: 点击图例可以开关每条线。鼠标悬停在 K 线上可以看到该日的详细数据。

### Q: 想查看更长时间范围的 K 线？
A: 底部的 DataZoom 滑块可以拖动选择时间范围，也可以缩放。
